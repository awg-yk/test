# 観測所ファインダー（Weather Station Finder）

気象庁の「過去の気象データ・ダウンロード」を、研究・解析向けに使いやすくするための非公式ツールです。
気象庁サイトの置き換えではなく、公開データをより効率よく取得するための補助ツールを目指します。

### 実データ整備の方法（フェーズ5）

観測所マスタは気象庁が公式に公開しているアメダス観測所表
（[amedastable.json](https://www.jma.go.jp/bosai/amedas/const/amedastable.json)、全国1,286地点）を元に、以下の手順で再構築しています（`build_stations.py`）。

1. **座標変換**: `[度, 分]` 形式の緯度経度を10進度に変換
2. **都道府県判定**: [dataofjapan/land](https://github.com/dataofjapan/land) の都道府県境界GeoJSON + `shapely` によるpoint-in-polygon判定で、
   緯度経度から都道府県を正確に特定（海岸線付近でポリゴン外に出てしまう17地点のみ、最近傍の都道府県で補完）
3. **地方の割り当て**: 判定した都道府県を、既存の地方マスタ（北海道・東北・関東・中部・近畿・中国・四国・九州・沖縄）で逆引き
4. **観測要素の判定**: `elems`（8桁: 気温・降水量・風向・風速・日照時間・積雪深・湿度・気圧の順）から、
   このアプリが扱う6要素（気温・降水量・風・積雪・湿度・日照時間）を導出（風向・風速はどちらか一方でも観測していれば「風」として扱う。気圧は対象外）
5. **種別の単純化**: `type: "A"`（気象官署）は「気象官署」、それ以外（B〜G、特別地域気象観測所・アメダス等）は「アメダス」として表示
6. **追加観測所のマージ**（フェーズ11）: `amedastable.json` は国内アメダス観測所の表なので南極・昭和基地を含まない。
   `scripts/extra_stations.json` に手書きで定義しておき、再生成のたびに追記する（地方「南極」もこのファイルで定義）。
   昭和基地は気象庁「過去の気象データ検索」(etrn) 上では「昭和（南極）」として `prec_no=99` / `block_no=89532` で公開されており、
   観測要素は気温・風・日照時間・積雪・湿度（降水量は地吹雪のため観測しておらず、etrnの日別値でも常に空欄のため含めていない）

全都道府県で1件以上のデータが確認できています（0件になった都道府県はなし）。処理スクリプト `scripts/build_stations.py` をリポジトリに同梱しているので、気象庁が観測所表を更新した際は同じ手順で再生成できます（`amedastable.json` を再取得し、`python3 scripts/build_stations.py` を実行）。

### 地域選択の動作仕様

- 何も選択しない状態 = 絞り込みなし（全観測所を表示）
- 地方チェックボックスをON/OFF → 配下の全都道府県が連動してON/OFF
- 都道府県を個別にON/OFFすると、所属する地方チェックボックスは自動的に
  「一部選択（indeterminate）」「全選択」「未選択」のいずれかに更新される
- 「すべて一括選択」チェックボックスも同様に、全体の選択状況を反映する
  （47都道府県に南極を加えた48地域が対象。南極には昭和基地1地点が属する）
- 「選択をクリア」ボタンで全解除できる
- 各項目の「(件数)」は、観測要素・種別・検索語による絞り込みを反映した件数を表示する（後述「件数表示の連動」）

### 観測要素フィルタの動作仕様

- 何も選択しない状態 = 絞り込みなし（全観測所を表示）
- 観測要素（気温・降水量・積雪・風・湿度・日照時間）を複数チェックすると、
  「すべて含む（AND）」「いずれか含む（OR）」を切り替えて絞り込める
- モードを切り替えても選択中の要素は維持される
- 「選択をクリア」ボタンで要素の選択のみ解除できる（モードは維持）

### 件数表示の連動（フェーズ11）

地域・観測要素・種別の各選択肢に付く「(件数)」は、**その軸自身の選択を無視し、他の軸の条件だけを適用した母集団**の件数を表示する（一般的なファセット検索と同じ数え方。`filterEngine.js` の `buildFacetCounts()`）。

- 地域で北海道だけを選ぶ → 観測要素・種別の件数が北海道内の件数になる
- 観測要素で積雪を選ぶ → 都道府県の件数が積雪観測地点だけの件数になる（例: 沖縄県は0件表示になり、薄いグレーで表示される）
- すでに選んでいる項目の件数が、自分自身の選択のせいで減って見えることはない
- キーワード検索は「軸」ではなく常時適用の条件として扱うため、すべての件数に効く
- 件数の更新はDOMのラベル文字列だけを差し替える方式で、チェック状態や地方グループの開閉状態は保持される

### 観測所一覧の動作仕様（フェーズ12で刷新）

- 列構成: 地点コード／地点名／かな／都道府県／地方／標高(m)／緯度／経度／観測要素／種別。
  標高・緯度・経度は全1,287地点で欠測が無く、地点同士を統一した基準で比較できる情報として表示している
  （緯度経度は小数第4位＝約10m精度まで表示）
- **地点名が気象庁「過去の気象データ検索」(etrn) へのリンク**になっており、クリックするとその地点を
  選択済みの状態でページが開く。年月とデータ種別（日別値・月別値・平年値など）はそのページで選ぶ
- 地点番号（`precNo`/`blockNo`）は全1,287地点で確定済みのため、全地点がこの地点別リンクになる
  （かつて7地点は新旧番号の候補が特定できず都道府県ページへの案内に留めていたが、フェーズ14で解消した）
- 観測要素は要素ごとに色分けしたタグで表示する（気温=赤／降水量=青／積雪=藍／風=緑／湿度=紫／日照時間=橙）。
  「観測要素で絞り込む」パネルの各項目に同じ色のドットを置いてあり、これが凡例を兼ねる

#### リンク先を etrn のトップ(index.php)にしている理由

日別値ページへの直リンク（`view/daily_s1.php`）は次の2点で全観測所共通の形にできず、実際にアメダス1,230地点で
「ページを表示することが出来ませんでした」になっていた（フェーズ12で判明・修正）。

- 気象官署は `daily_s1.php`、アメダスは `daily_a1.php` と種別ごとにファイルが異なる
- `year` / `month` を空にするとどちらの形式でもエラーページになる

`stats/etrn/index.php?prec_no=..&block_no=..` は種別に関係なく地点が選択された状態で開き、年月・データ種別を
そこから選べるため、こちらを採用している。

### CSVエクスポート・気象庁公式ページの動作仕様

- 「選択結果をCSVでダウンロード」ボタンで、**現在の絞り込み条件で表示中の観測所一覧**をCSVとして保存できる
  （列: 観測所ID・地点名・かな・都道府県・地方・緯度・経度・標高(m)・種別・観測要素・気象庁ページURL）

  （調査結果。詳細は `js/modules/exporter.js` のコメント参照）

### 地図表示の動作仕様

- 地域・観測要素・検索の絞り込み結果（一覧テーブルと同じ`visibleStations`）を地図上にマーカー表示する
- マーカーの色は種別で区別: 気象官署=青緑、アメダス=橙（凡例をパネル下部に表示）
- マーカーをクリックすると、地点名・かな・都道府県・種別・標高・緯度経度・地点コード・観測要素・気象庁リンクをポップアップ表示する
  （気象庁リンクは一覧と同じ扱い。地点番号は全地点で確定済みのため、常に地点別リンクを表示する）
- **地図の拡大縮小・移動には Ctrl（macOSは⌘）キーが必要**（フェーズ11）
  - Ctrl＋ホイールで拡大縮小、Ctrl＋ドラッグで移動。Ctrlを押していないときはページが通常どおりスクロールし、地図の上に操作ヒントを一時表示する
  - タッチ端末では1本指の操作はページのスクロール、2本指で地図の拡大縮小・移動になる
  - Ctrl無しで使える操作: 左上の ＋／− ボタン、ダブルクリックでの拡大、右上の「検索結果に合わせる」ボタン、地図フォーカス中の矢印キー
  - 実装上は Leaflet の `scrollWheelZoom` / `dragging` を無効にしたうえで、修飾キーの押下状態に応じて自前でズーム（`setZoomAround`）とドラッグ有効化を行っている（「Ctrl検知後にLeafletのハンドラを有効化」ではその回のイベントがハンドラに届かず、1回目だけブラウザのページ拡大が起きてしまうため）
- 絞り込み条件を変えると、表示範囲（ズーム・中心）は対象観測所が収まるよう自動調整される。
  ただし**絞り込みが無い（全観測所表示）ときは表示範囲を動かさない**（南極まで含めて範囲を合わせると地図が世界全体まで引いてしまうため。日本全体が収まる初期表示のままにする）
- 右上の「検索結果に合わせる」ボタンで、いつでも現在の絞り込み結果全体が収まる位置に戻せる
- 全国選択時など表示件数が多い場合は、近接するマーカーをクラスタ化して表示する（Leaflet.markercluster使用。ズームすると自動的に個別マーカーに分解される）
- ページネーションでページを送っても地図は再描画されない（地図は「絞り込み結果全体」を表示する仕様のため、テーブルのページングとは独立している）
- Leaflet / Leaflet.markercluster は`vendor/leaflet/`にリポジトリ同梱しており、外部CDNには依存しない（フェーズ18）。`vendor/leaflet/`が欠けている・壊れている場合のみ地図パネルにエラーメッセージを表示する（他機能には影響しない）
- **マーカー⇔一覧行の相互連携**（フェーズ15）: 地図のマーカーをクリックすると、対応する一覧の行がある位置までページが自動的に切り替わり、その行がハイライト＆スクロール表示される。逆に一覧の行をクリックすると、地図がその地点へパンしてポップアップが開く（クラスタ化されている場合はクラスタを解いてから開く）。マーカーと行はどちらも`store.selectedStationId`という共通の状態を介してやり取りしており、絞り込み条件を変えたり手動でページを送ったりすると選択状態は解除される

### URLクエリ化の動作仕様

- 地域（`pref`）・観測要素（`elem`）・AND/OR（`mode`）・検索語（`q`）・ページ番号（`page`）を、絞り込みのたびに`history.replaceState`でURLへ反映する（`pushState`は使わないため、フィルタ操作のたびに「戻る」履歴が積み上がることはない）
- 既定値のパラメータ（絞り込みなし・AND・1ページ目）は省略され、URLはできるだけ短く保たれる。何も絞り込んでいない状態はパラメータの付かないトップページURLと同じになる
- ステータスバーの「この条件のURLをコピー」ボタンで、現在表示中のURL（絞り込み条件込み）をクリップボードにコピーできる
- 上記URLで直接アクセスすると、ページ読み込み時にクエリを解析し、地域選択・観測要素選択・AND/OR・検索ボックス・ページ番号・一覧・地図のすべてに同じ条件を復元する
- 都道府県名や検索語に含まれる日本語は`URLSearchParams`が自動的にパーセントエンコードする（ブラウザのアドレスバー上では大抵の場合デコードされた日本語表示になる）

### 種別フィルタの動作仕様

- 「種別で絞り込む」パネルで気象官署／アメダスを選択できる。地域・観測要素とは独立した絞り込み軸で、他の条件とはAND合成される（何も選択しない場合は絞り込みなし＝両方表示、他の絞り込みUIと同じ方針）
- URLクエリでは`type=kansho,amedas`のように短いコードで表現される

> フェーズ9で追加した「プリセットで絞り込む」パネル（気象官署のみ／積雪観測地点のみ 等）は、
> 個別の絞り込みパネルで同じことができ画面を占有するだけだったため、フェーズ12で削除した
> （`js/modules/presets.js` / `presetPanel.js` / `test-presets.mjs` も併せて削除）。

### 廃止済み観測所の動作仕様（フェーズ16・17・19）

- 「廃止済み観測所」パネルのチェックボックスをオンにすると、現在は運用されていない過去の観測所506地点が
  一覧・地図・件数集計に合流する。既定ではオフ（＝一覧・地図には現役の1,287地点のみを表示）
- 廃止済み観測所は`data/stations.json`の`discontinuedStations`に、現役の`stations`とは別の配列として収録している。
  地域・観測要素・種別・検索のいずれの絞り込みも、チェックがオンのときだけこの506地点を母集団に含める
  （オフのときは他の絞り込みUIの件数・一覧・地図から完全に除外される）
- 一覧では地点名の隣に「廃止」タグを表示し、その下に観測期間（例:「観測期間: 1961年1月〜2002年2月」）を表示する。
  precNo/blockNoが確定している361地点は地点名が現役観測所と同じように気象庁ページへのリンクになる
  （フェーズ19）。残る139地点は地点番号が未確定のためリンクにならない
- 地図ではマーカーの色を種別に関わらずグレーで統一表示する（凡例に「廃止済み」を追加）。ポップアップには
  常に観測期間を表示し、precNo/blockNoが確定していれば気象庁リンクも併記する。未確定の場合はリンクの
  代わりに「地点番号未確定のためリンクはありません」と案内する
- CSVエクスポートには「状態」（現役／廃止済み）と「観測期間」の列を追加している
- チェック状態はURLクエリ`discontinued=1`に反映され、共有リンクからも復元できる
- データの出どころ: 気象官署・測候所12地点は気象庁の観測所履歴ファイル`smaster.index`（気象庁側に同名で
  複数の地点番号が存在するのと同様、アメダス網の外にある独立した観測所のため後述のアメダス履歴には出てこない）。
  アメダス494地点は気象庁純正の**アメダス地点情報履歴ファイル`amdmaster.index`**（全履歴7,935レコード）を
  解析し、番号の付け替え（同一地点が新しい番号で運用を続けているケース）を`Old Station Number`列で
  追跡したうえで、本当に運用を終えた地点だけを抽出している。緯度経度・標高・観測要素（気温/降水量/風/
  日照時間/積雪/湿度）は同ファイルの記載値をそのまま使用しており、都道府県の判定は`scripts/build_stations.py`
  と同じ方式（境界GeoJSON + point-in-polygon）で行っている

### 検索・ページネーションの動作仕様

- 検索ボックスに入力すると、地点名・かな・英語名・都道府県名のいずれかに部分一致する観測所だけに絞り込まれる
  （大文字小文字は区別しない。入力が止まってから200ms後に反映されるデバウンス処理付き）
- 検索・地域選択・観測要素フィルタはすべて組み合わせて使える（AND条件で合成）
- 一覧は50件ずつページ分割して表示する。地域・観測要素・検索条件を変更すると、自動的に1ページ目に戻る
- テーブルヘッダーは縦スクロールしても常に見える位置に固定表示される

## データについて（重要）

`data/stations.json` は **プロトタイプ用のサンプルデータ** です。
各都道府県の代表的な観測地点を1件ずつ、大まかな座標・観測要素で収録していますが、
実運用前に気象庁の最新の観測所一覧と照合し、正確なデータに差し替える必要があります。

このツールは気象庁の生観測データを再配布するものではなく、観測所を探すための
メタデータ（地点名・地方・都道府県・観測要素など）のみを扱う設計です。
実際の気象データ取得は、最終的に気象庁の該当ページへ案内する形を想定しています。
